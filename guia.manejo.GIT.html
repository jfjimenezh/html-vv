Que es git hub?

Git es una plataforma de alojamiento de codigo para el control de versiones y la colaboracion, permite trabajar en conjunto en proyectos desde cualquier lugar.

Repositorio GITHUB

Un repositorio se usa para organizar proyectos, estos pueden contener archivos, carpetas, imagenes, hojas de calculo y videos, generalmente se incluye al crear un repositorio un archivo readme.md el cual se usa para incluir la informacion sobre su proyecto.Se describira una guia paso a paso para crear un repositorio y manejar GIT mediante una lista odenada:

  1. Se crea una cuenta el GUITHUB click y podras crear tu primer repositorio
  2. Se crea una carpeta donde se desee dentro de tu ordenador con el nombre de tu preferencia.
  3. Ingresas al link anteriorente dicho con tu respectiva cuenta y en la parte superior derecha encontraras el simbolo de adicion y haciendo click en la opcion crear repositorio.
  4. Se abre visual studio code y desde alli se llama el folder que se creo en el PC anteriormente.
  5. En la terminal de visual se digita el siguiente comando:
      echo "nombre de la carpeta">>Readme.md
  6. Se inicializa el reporitorio mediante el comando "git init"
  7. Se pueden realizar pines y estos datos guardarlos en una carpeta con el siguiente comando: ping google.com >> nombre archivo.txt
  8. En la opcion source control se pueden verificar los cambios que se realizaron.
  9. Se realiza el primer commit el cual es un mensaje de confirmacion asociado a un cambio y es una descripcion de el por que se realizo este cambio, mediante el siguiente comando en la terminal. git commit -m "mensaje que desee" o mediante las opcones de visual.
  10. se digita el siguiente comando "git remote add origin "la direccion del repositorio"
  11. Se envian los datos a la nuve mediante "git push"
  12. Podemos crear una rama y saltar a ella mediante el comando "git checkout -b ombre de rama"
  13. Podemos ver en que rama estamos trabajando actualmente mediante "git branch"
  14. Podemos crear un nuevo documento en la opcion Explorer > New file > nombre del archivo con su extension, trabajar en ello y subir la informacion a la nuve como anteriormente de dijo.
  15. Se pueden agregar archivos antes de subirlos ya que arrojaria error mediante el comando "git add ."
  16. Es necesario actualizar los cambios del remoto a los archivos locales mediante " git pull"
  17. Existe una forma de revisar el estado de mis archivos y saber si estan actualizados mediante el comando "git status"
  18. Se realiza una solicitud de extraccion la cual propone sus cambios y solicita que sean revisados para extraer su contribucion y sea combinada en su rama.Esto se realiza en la pagina git en la opcion "pull request".
  19. Por ultimo se realiza una combinacion la cual combina la solicitud de extraccion con la rama principal "Master".
